🍎 WallFruits Backend Deploy

Guia passo a passo para fazer deploy no Render

1 Preparação: Supabase

Primeiro, você precisa ter as credenciais do Supabase. Vamos coletar 4 informações importantes:

📋 Passo 1a: Encontre DATABASE_URL
  1. Acesse: Supabase Dashboard
  2. Clique em seu projeto
  3. Vá em Settings → Database
  4. Copie a Connection String (a que começa com postgresql://)
  5. A URL deve ter formato: postgresql://postgres.xxxxx:senha@aws-0-us-east-1.pooler.supabase.com:6543/postgres?sslmode=require
📋 Passo 1b: Encontre SUPABASE_URL
  1. Ainda em Settings → Database
  2. Procure por "Project URL"
  3. Copie a URL (formato: https://seu-projeto.supabase.co)
📋 Passo 1c & 1d: Encontre as KEYS
  1. Vá em Settings → API
  2. Copie "anon public" → isso é sua SUPABASE_ANON_KEY
  3. Copie "service_role secret" → isso é sua SUPABASE_SERVICE_ROLE_KEY
Valores que você deve ter agora: DATABASE_URL postgresql://... SUPABASE_URL https://seu-projeto.supabase.co SUPABASE_ANON_KEY eyJhbGc... (string longa) SUPABASE_SERVICE_ROLE_KEY eyJhbGc... (string longa, diferente da anterior)

2 Acesse o Render Dashboard

Agora vamos configurar essas variáveis no Render:

  1. Acesse: Render Dashboard
  2. Você está logado? ✓ Perfeito!
  3. Localize o serviço "wallfruits-api"
  4. Clique nele para abrir

3 Configurar Variáveis de Ambiente

No painel do serviço, procure pela aba "Environment" ou "Environment Variables"

⚠️ Cuidado: Não compartilhe suas KEYS com ninguém! Elas são secrets!

Clique em "Add Environment Variable" e configure cada uma:

Variável 1:
Key: DATABASE_URL
Value: Cole aqui o postgresql://... do Supabase
Variável 2:
Key: SUPABASE_URL
Value: https://seu-projeto.supabase.co
Variável 3:
Key: SUPABASE_ANON_KEY
Value: Cole a "anon public" do Supabase
Variável 4:
Key: SUPABASE_SERVICE_ROLE_KEY
Value: Cole a "service_role secret" do Supabase
Variável 5:
Key: SUPABASE_AUTH_ENABLED
Value: true
Variável 6:
Key: DEBUG
Value: false
Variável 7:
Key: REDIS_ENABLED
Value: false
✅ Depois de preencher TODAS as 7 variáveis:
  1. Clique em "Save Changes" (botão no final da página)
  2. Aguarde a página recarregar

4 Fazer Deploy

Agora vamos deploiar o código com essas variáveis:

  1. No painel do serviço, procure o botão "Manual Deploy" (ou "Deploy")
  2. Clique nele
  3. Selecione "Deploy latest commit"
  4. Aguarde o build (vai mostrar logs)
⏱️ Tempo de espera: O build leva entre 3-7 minutos. Não feche a página!

Você vai ver:

  • Logs de build (muitas linhas de texto)
  • Um ponto colorido em cima do serviço passando por: updating → deploying → Live ✅

5 Testar o Deploy

Quando o status ficar "Live" (verde), teste sua API:

Teste 1: Health Check
Abra no navegador:
https://wallfruits-api.onrender.com/health

Resultado esperado: Uma página com {"status":"ok"}

Teste 2: Swagger API Docs
Abra no navegador:
https://wallfruits-api.onrender.com/docs

Resultado esperado: Interface Swagger com todos seus endpoints

🎉 Parabéns! Se ambas as URLs funcionaram, sua API está no ar e pronta para usar!

❓ Dúvidas Comuns

P: O deploy deu erro de "DATABASE_URL"
R: Copie a URL com ?sslmode=require no final. Não esqueça desta parte!
P: Os logs mostram "ConnectionError"
R: Verifique se a DATABASE_URL está correta e se o PostgreSQL do Supabase está rodando.
P: A página /docs carrega mas está vazia
R: Pode ser cache do navegador. Tente Ctrl+F5 ou abra em modo anônimo.
P: Quero fazer outro deploy após atualizar código
R: Faça git push no GitHub, depois clique "Manual Deploy" → "Deploy latest commit" no Render.
